My Flix app

This web app provides users with access to movie details, directors, and genres. Users can register, update their info, and create a list of favorite movies.

MyFlix API Documentation

Business Logic URL HTTP Method Request body data format Response body data format
Get data about a single movie by title /movies/:title GET None A JSON object holding all info data of a single movie, containing title, year, image url, a brief description, director info and the genre. Example: { "Title": "Inception", "Description": "A thief who steals corporate secrets through the use of dream-sharing technology is given the inverse task of planting an idea into the mind of a C.E.O.", "Genre": { "Name": "Sci-Fi", "Description": "Science fiction films are speculative fiction that explore imaginative and futuristic concepts." }, "Director": { "Name": "Christopher Nolan", "Bio": "Christopher Nolan, a British-American film director, producer, and screenwriter born in 1970, is known for his innovative storytelling and groundbreaking films.", "Birth": "1970", "Death": null }, "Featured": true }
Return data about a genre by name /movies/genre/:genreName GET None A JSON object holding data about a single genre, containing genre name and a brief description. Example: "Genre": { "Name": "Sci-Fi", "Description": "Science fiction films are speculative fiction that explore imaginative and futuristic concepts." }
Return data with info about a specific director /movies/directors/:directorsName GET None A JSON object holding data about a single director, containing the director's name, birth year, year of death (if applicable) and a brief bio. Example: "Director": { "Name": "Christopher Nolan", "Bio": "Christopher Nolan, a British-American film director, producer, and screenwriter born in 1970, is known for his innovative storytelling and groundbreaking films.", "Birth": "1970", "Death": null }
Allow new users to register /users POST A JSON object holding data about the user to add, structured as follows: { "Username": "newUser", "Password": "password123", "Email": "newuser@example.com", "Birthday": "1990-01-01T00:00:00.000Z", "FavoriteMovies": [], "_id": "6565fdb3fc8d5f4e857f5dd2", "__v": 0 } A JSON object holding user data plus an ID: favorite movies: { "_id": "6565fdb3fc8d5f4e857f5dd2", "Username": "newUser", "Password": "newPassword123", "Email": "new.email@example.com", "Birthday": "1990-01-01T00:00:00.000Z", "FavoriteMovies": [ "656495341a40824464f3e0b5" ], "__v": 1 }
Allow users to update their user info (username, password, email, date of birth) /users/:username PUT A JSON object holding the user info that is subject to change. It should contain at least one of the following key/value pairs: { "_id": "6565fdb3fc8d5f4e857f5dd2", "Username": "newUser", "Password": "newPassword123", "Email": "new.email@example.com", "Birthday": "1990-01-01T00:00:00.000Z", "FavoriteMovies": [], "__v": 0 } A JSON object holding data about the updated user information. Example: { "_id": "6565fdb3fc8d5f4e857f5dd2", "Username": "newUser", "Password": "newPassword123", "Email": "new.email@example.com", "Birthday": "1990-01-01T00:00:00.000Z", "FavoriteMovies": [], "__v": 0 }
Allow users to add a movie to their list of favorites /users/:Username/movies/:movieID POST None A JSON object holding all user info with all key/values pair updated: { "_id": "6565fdb3fc8d5f4e857f5dd2", "Username": "newUser", "Password": "newPassword123", "Email": "new.email@example.com", "Birthday": "1990-01-01T00:00:00.000Z", "FavoriteMovies": [ "656495341a40824464f3e0b5" ], "__v": 1 }
Allow users to remove a movie from their list of favorites /users/:Username/movies/:movieID DELETE None A JSON object holding all user info with all key/values pair updated: { "_id": "6565fdb3fc8d5f4e857f5dd2", "Username": "newUser", "Password": "newPassword123", "Email": "new.email@example.com", "Birthday": "1990-01-01T00:00:00.000Z", "FavoriteMovies": [ "656495341a40824464f3e0b5" ], "__v": 1 }
Delete user registration /users/:Username DELETE None A text message indicating whether a user was successfully removed.
Allow users to update their password /users/:username/password PUT A JSON object holding the user's new password. Example: { "password": "newpassword456" } A JSON object indicating the success of the password update. Example: { "message": "Password updated successfully." }
